# Quick Start

Get Everything Claude Code installed and ready to use in under 2 minutes.

<Steps>
  <Step title="Install the Plugin">
    The easiest way to use Everything Claude Code is to install it as a Claude Code plugin.
    
    <CodeGroup>
    ```bash Commands
    # Add marketplace
    /plugin marketplace add affaan-m/everything-claude-code
    
    # Install plugin
    /plugin install everything-claude-code@everything-claude-code
    ```
    
    ```json Manual (settings.json)
    {
      "extraKnownMarketplaces": {
        "everything-claude-code": {
          "source": {
            "source": "github",
            "repo": "affaan-m/everything-claude-code"
          }
        }
      },
      "enabledPlugins": {
        "everything-claude-code@everything-claude-code": true
      }
    }
    ```
    </CodeGroup>
    
    This gives you instant access to:
    - **13 specialized agents** (planner, architect, tdd-guide, code-reviewer, etc.)
    - **33 slash commands** (`/plan`, `/tdd`, `/code-review`, `/build-fix`, etc.)
    - **56+ skills** (coding standards, TDD workflow, security review, etc.)
    - **Automated hooks** (memory persistence, strategic compaction, security warnings)
  </Step>
  
  <Step title="Install Rules (Required)">
    <Warning>
    Claude Code plugins cannot distribute `rules` automatically ([upstream limitation](https://code.claude.com/docs/en/plugins-reference)). You need to install rules manually.
    </Warning>
    
    Clone the repository first:
    
    ```bash
    git clone https://github.com/affaan-m/everything-claude-code.git
    cd everything-claude-code
    ```
    
    <Tabs>
      <Tab title="Automated (Recommended)">
        Use the installer script to handle common + language rules safely:
        
        <CodeGroup>
        ```bash TypeScript
        ./install.sh typescript
        ```
        
        ```bash Python
        ./install.sh python
        ```
        
        ```bash Go
        ./install.sh golang
        ```
        
        ```bash Multiple Languages
        ./install.sh typescript python golang
        ```
        
        ```bash Cursor IDE
        ./install.sh --target cursor typescript
        ```
        </CodeGroup>
        
        The installer copies rules to `~/.claude/rules/` maintaining the directory structure:
        - `common/` - Language-agnostic principles (always installed)
        - `typescript/`, `python/`, `golang/`, etc. - Language-specific rules
      </Tab>
      
      <Tab title="Manual">
        For manual installation, copy rules maintaining the directory structure:
        
        <CodeGroup>
        ```bash User-level (all projects)
        # Common rules (always install)
        mkdir -p ~/.claude/rules
        cp -r rules/common/* ~/.claude/rules/
        
        # Language-specific rules (pick your stack)
        cp -r rules/typescript/* ~/.claude/rules/
        cp -r rules/python/* ~/.claude/rules/
        cp -r rules/golang/* ~/.claude/rules/
        ```
        
        ```bash Project-level (current project only)
        # Common rules (always install)
        mkdir -p .claude/rules
        cp -r rules/common/* .claude/rules/
        
        # Language-specific rules (pick your stack)
        cp -r rules/typescript/* .claude/rules/
        ```
        </CodeGroup>
      </Tab>
    </Tabs>
    
    <Info>
    Rules are always-follow guidelines that ensure consistent code quality, security, and best practices across your projects.
    </Info>
  </Step>
  
  <Step title="Start Using Commands">
    Try your first command to verify installation:
    
    <CodeGroup>
    ```bash Plugin Install (Namespaced)
    # Plan a new feature
    /everything-claude-code:plan "Add user authentication"
    
    # Run TDD workflow
    /everything-claude-code:tdd
    
    # Review code changes
    /everything-claude-code:code-review
    
    # Check available commands
    /plugin list everything-claude-code@everything-claude-code
    ```
    
    ```bash Manual Install (Shorter Form)
    # Plan a new feature
    /plan "Add user authentication"
    
    # Run TDD workflow
    /tdd
    
    # Review code changes
    /code-review
    ```
    </CodeGroup>
    
    <Note>
    Plugin installs use the namespaced form (`/everything-claude-code:plan`) while manual installs use the shorter form (`/plan`).
    </Note>
  </Step>
  
  <Step title="Configure Package Manager (Optional)">
    The plugin automatically detects your preferred package manager (npm, pnpm, yarn, bun) with the following priority:
    
    1. Environment variable: `CLAUDE_PACKAGE_MANAGER`
    2. Project config: `.claude/package-manager.json`
    3. `package.json`: `packageManager` field
    4. Lock file detection
    5. Global config: `~/.claude/package-manager.json`
    6. Fallback: First available
    
    To set your preferred package manager:
    
    <CodeGroup>
    ```bash Environment Variable
    export CLAUDE_PACKAGE_MANAGER=pnpm
    ```
    
    ```bash Global Config
    node scripts/setup-package-manager.js --global pnpm
    ```
    
    ```bash Project Config
    node scripts/setup-package-manager.js --project bun
    ```
    
    ```bash Detect Current
    node scripts/setup-package-manager.js --detect
    ```
    
    ```bash Claude Code Command
    /setup-pm
    ```
    </CodeGroup>
  </Step>
</Steps>

## Common First Commands

| Command | What It Does | When to Use |
|---------|--------------|-------------|
| `/plan "feature"` | Creates implementation plan | Starting a new feature |
| `/tdd` | Enforces test-driven development | Writing new code or fixing bugs |
| `/code-review` | Reviews code for quality & security | After writing/modifying code |
| `/build-fix` | Fixes build and type errors | When build fails |
| `/e2e` | Generates E2E tests | Testing critical user flows |
| `/orchestrate` | Multi-agent workflow | Complex tasks requiring multiple agents |

## Example Workflow

Here's a typical workflow for adding a new feature:

<Steps>
  <Step title="Plan the Feature">
    ```bash
    /everything-claude-code:plan "Add OAuth authentication with Google and GitHub"
    ```
    The planner agent creates a detailed implementation blueprint.
  </Step>
  
  <Step title="Implement with TDD">
    ```bash
    /everything-claude-code:tdd
    ```
    The tdd-guide agent enforces write-tests-first workflow.
  </Step>
  
  <Step title="Review the Code">
    ```bash
    /everything-claude-code:code-review
    ```
    The code-reviewer agent checks quality, security, and maintainability.
  </Step>
  
  <Step title="Fix Any Build Errors">
    ```bash
    /everything-claude-code:build-fix
    ```
    The build-error-resolver agent fixes TypeScript errors and build issues.
  </Step>
  
  <Step title="Add E2E Tests">
    ```bash
    /everything-claude-code:e2e
    ```
    The e2e-runner agent generates Playwright tests for critical flows.
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Commands not found / plugin not loading">
    Verify the plugin is enabled in `~/.claude/settings.json`:
    
    ```json
    {
      "enabledPlugins": {
        "everything-claude-code@everything-claude-code": true
      }
    }
    ```
    
    Restart Claude Code after enabling the plugin.
  </Accordion>
  
  <Accordion title="Hooks aren't working / 'Duplicate hooks file' errors">
    **Do NOT add a `"hooks"` field to `.claude-plugin/plugin.json`.**
    
    Claude Code v2.1+ automatically loads `hooks/hooks.json` from installed plugins. Explicitly declaring it causes duplicate detection errors.
    
    See issues [#29](https://github.com/affaan-m/everything-claude-code/issues/29), [#52](https://github.com/affaan-m/everything-claude-code/issues/52), [#103](https://github.com/affaan-m/everything-claude-code/issues/103).
  </Accordion>
  
  <Accordion title="Context window shrinking / running out of context">
    Too many MCP servers eat your context. Each MCP tool description consumes tokens from your 200k window, potentially reducing it to ~70k.
    
    **Fix:** Disable unused MCPs per project in `.claude/settings.json`:
    
    ```json
    {
      "disabledMcpServers": ["supabase", "railway", "vercel"]
    }
    ```
    
    Keep under 10 MCPs enabled and under 80 tools active.
  </Accordion>
  
  <Accordion title="Rules not being followed">
    Verify rules are installed in the correct location:
    
    ```bash
    # Check user-level rules
    ls -la ~/.claude/rules/
    
    # Check project-level rules
    ls -la .claude/rules/
    ```
    
    Rules should maintain the directory structure:
    - `common/` - Language-agnostic principles
    - `typescript/`, `python/`, `golang/`, etc. - Language-specific rules
  </Accordion>
</AccordionGroup>
