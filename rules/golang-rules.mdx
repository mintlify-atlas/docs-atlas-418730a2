## Coding Style

### Formatting

<Card title="gofmt and goimports are mandatory" icon="code">
  No style debates — the Go toolchain enforces formatting
</Card>

```bash
# Format all Go files
gofmt -w .
goimports -w .
```

### Design Principles

<AccordionGroup>
  <Accordion title="Accept interfaces, return structs">
    Functions should accept interfaces (flexible) and return concrete types (specific).
    
    ```go
    // GOOD
    func ProcessUser(r UserRepository) *User {
      // r is an interface, return value is concrete
    }
    
    // AVOID
    func ProcessUser(db *Database) UserInterface {
      // Both are too specific/abstract
    }
    ```
  </Accordion>
  
  <Accordion title="Keep interfaces small (1-3 methods)">
    Small, focused interfaces are easier to implement and test.
    
    ```go
    // GOOD: Small, focused interface
    type Reader interface {
      Read(p []byte) (n int, err error)
    }
    
    // AVOID: Large interface
    type UserManager interface {
      Create(...) error
      Update(...) error
      Delete(...) error
      Find(...) (*User, error)
      List(...) ([]*User, error)
      Validate(...) error
      // ... many more methods
    }
    ```
  </Accordion>
</AccordionGroup>

### Error Handling

Always wrap errors with context:

```go
if err != nil {
    return fmt.Errorf("failed to create user: %w", err)
}
```

<Info>
  Use `%w` to wrap errors (Go 1.13+) for error unwrapping with `errors.Is` and `errors.As`
</Info>

## Testing

### Framework

Use the standard `go test` with **table-driven tests**.

```go
func TestCalculateTotal(t *testing.T) {
    tests := []struct {
        name     string
        input    []int
        expected int
    }{
        {"empty slice", []int{}, 0},
        {"single item", []int{5}, 5},
        {"multiple items", []int{1, 2, 3}, 6},
    }
    
    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            result := CalculateTotal(tt.input)
            if result != tt.expected {
                t.Errorf("got %d, want %d", result, tt.expected)
            }
        })
    }
}
```

### Race Detection

Always run with the `-race` flag:

```bash
go test -race ./...
```

<Warning>
  Race conditions can cause subtle, hard-to-debug issues in production. Always test with `-race` during development.
</Warning>

### Coverage

```bash
go test -cover ./...
```

## Patterns

### Functional Options

```go
type Option func(*Server)

func WithPort(port int) Option {
    return func(s *Server) { s.port = port }
}

func NewServer(opts ...Option) *Server {
    s := &Server{port: 8080}
    for _, opt := range opts {
        opt(s)
    }
    return s
}

// Usage
server := NewServer(
    WithPort(3000),
    WithTimeout(30*time.Second),
)
```

### Small Interfaces

Define interfaces where they are used, not where they are implemented.

```go
// GOOD: Define interface in consumer package
package service

type UserRepository interface {
    FindByID(id string) (*User, error)
}

type UserService struct {
    repo UserRepository
}

// Implementation in separate package
package postgres

type PostgresRepo struct { ... }

func (r *PostgresRepo) FindByID(id string) (*User, error) { ... }
```

### Dependency Injection

Use constructor functions to inject dependencies:

```go
func NewUserService(repo UserRepository, logger Logger) *UserService {
    return &UserService{repo: repo, logger: logger}
}
```

## Security

### Secret Management

```go
apiKey := os.Getenv("OPENAI_API_KEY")
if apiKey == "" {
    log.Fatal("OPENAI_API_KEY not configured")
}
```

### Security Scanning

Use **gosec** for static security analysis:

```bash
gosec ./...
```

### Context & Timeouts

Always use `context.Context` for timeout control:

```go
ctx, cancel := context.WithTimeout(ctx, 5*time.Second)
defer cancel()

resp, err := client.DoWithContext(ctx, req)
if err != nil {
    return fmt.Errorf("request failed: %w", err)
}
```

<Info>
  Timeouts prevent goroutines from hanging indefinitely and consuming resources
</Info>

## Hooks

### PostToolUse Hooks

Configure in `~/.claude/settings.json`:

<AccordionGroup>
  <Accordion title="gofmt/goimports: Auto-format .go files after edit">
    Automatically runs gofmt and goimports on Go files after edits.
    
    ```json
    {
      "matcher": "Edit",
      "hooks": [{
        "type": "command",
        "command": "node format-go.js"
      }]
    }
    ```
    
    Example hook:
    ```javascript
    const { execFileSync } = require('child_process');
    
    let data = '';
    process.stdin.on('data', chunk => data += chunk);
    process.stdin.on('end', () => {
      const input = JSON.parse(data);
      const filePath = input.tool_input?.file_path || '';
      
      if (/\.go$/.test(filePath)) {
        try {
          execFileSync('goimports', ['-w', filePath], { stdio: 'pipe' });
        } catch (e) {}
      }
      
      console.log(data);
    });
    ```
  </Accordion>
  
  <Accordion title="go vet: Run static analysis after editing .go files">
    Runs `go vet` to catch common mistakes.
    
    ```bash
    go vet ./...
    ```
  </Accordion>
  
  <Accordion title="staticcheck: Run extended static checks on modified packages">
    Runs staticcheck for more comprehensive analysis.
    
    ```bash
    staticcheck ./...
    ```
  </Accordion>
</AccordionGroup>
