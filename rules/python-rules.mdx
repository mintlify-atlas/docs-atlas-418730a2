## Coding Style

### Standards

- Follow **PEP 8** conventions
- Use **type annotations** on all function signatures

### Immutability

Prefer immutable data structures:

<CodeGroup>
```python Frozen dataclass
from dataclasses import dataclass

@dataclass(frozen=True)
class User:
    name: str
    email: str
```

```python NamedTuple
from typing import NamedTuple

class Point(NamedTuple):
    x: float
    y: float
```
</CodeGroup>

## Testing

### Framework

Use **pytest** as the testing framework.

```bash
pytest --cov=src --cov-report=term-missing
```

### Test Organization

Use `pytest.mark` for test categorization:

```python
import pytest

@pytest.mark.unit
def test_calculate_total():
    assert calculate_total([1, 2, 3]) == 6

@pytest.mark.integration
def test_database_connection():
    db = Database()
    assert db.connect() is True
```

## Patterns

### Protocol (Duck Typing)

```python
from typing import Protocol

class Repository(Protocol):
    def find_by_id(self, id: str) -> dict | None: ...
    def save(self, entity: dict) -> dict: ...
```

### Dataclasses as DTOs

```python
from dataclasses import dataclass

@dataclass
class CreateUserRequest:
    name: str
    email: str
    age: int | None = None
```

### Context Managers & Generators

<AccordionGroup>
  <Accordion title="Context Managers">
    Use context managers (`with` statement) for resource management:
    
    ```python
    with open('file.txt', 'r') as f:
        content = f.read()
    
    # Custom context manager
    from contextlib import contextmanager
    
    @contextmanager
    def transaction(db):
        try:
            yield db
            db.commit()
        except Exception:
            db.rollback()
            raise
    ```
  </Accordion>
  
  <Accordion title="Generators">
    Use generators for lazy evaluation and memory-efficient iteration:
    
    ```python
    def fibonacci(n):
        a, b = 0, 1
        for _ in range(n):
            yield a
            a, b = b, a + b
    
    # Memory-efficient iteration over large datasets
    for num in fibonacci(1000000):
        if num > 100:
            break
    ```
  </Accordion>
</AccordionGroup>

## Hooks

### PostToolUse Hooks

Configure in `~/.claude/settings.json`:

<AccordionGroup>
  <Accordion title="black/ruff: Auto-format .py files after edit">
    Automatically runs black or ruff formatter on Python files after edits.
    
    ```json
    {
      "matcher": "Edit",
      "hooks": [{
        "type": "command",
        "command": "node format-python.js"
      }]
    }
    ```
    
    Example hook:
    ```javascript
    const { execFileSync } = require('child_process');
    
    let data = '';
    process.stdin.on('data', chunk => data += chunk);
    process.stdin.on('end', () => {
      const input = JSON.parse(data);
      const filePath = input.tool_input?.file_path || '';
      
      if (/\.py$/.test(filePath)) {
        try {
          execFileSync('ruff', ['format', filePath], { stdio: 'pipe' });
        } catch (e) {}
      }
      
      console.log(data);
    });
    ```
  </Accordion>
  
  <Accordion title="mypy/pyright: Run type checking after editing .py files">
    Runs type checker to catch type errors immediately.
    
    ```bash
    mypy src/
    # or
    pyright src/
    ```
  </Accordion>
</AccordionGroup>

### Warnings

<Warning>
  Warn about `print()` statements in edited files (use `logging` module instead)
</Warning>

```python
import logging

logger = logging.getLogger(__name__)

# WRONG: print() in production code
print(f"Processing user {user_id}")

# CORRECT: Use logging
logger.info(f"Processing user {user_id}")
```
