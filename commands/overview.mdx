## Command Categories

### Planning & Architecture

Commands for planning features and making architectural decisions:

- **[/plan](/commands/plan)** - Restate requirements, assess risks, create implementation plan
- **[/orchestrate](/commands/orchestrate)** - Sequential agent workflow (planner → tdd → reviewer → security)
- **[/multi-plan](/commands/multi-plan)** - Multi-model collaborative planning with Codex and Gemini

### Test-Driven Development

Commands that enforce testing best practices:

- **[/tdd](/commands/tdd)** - Scaffold interfaces, write tests first, implement minimal code
- **[/go-test](/commands/go-test)** - Go TDD with table-driven tests
- **[/test-coverage](/commands/test-coverage)** - Analyze coverage gaps and generate missing tests
- **[/e2e](/commands/e2e)** - Generate and run Playwright end-to-end tests

### Code Quality & Review

Commands for maintaining code quality:

- **[/code-review](/commands/code-review)** - Security and quality review of uncommitted changes
- **[/go-review](/commands/go-review)** - Idiomatic Go code review with concurrency analysis
- **[/python-review](/commands/python-review)** - PEP 8 compliance and Pythonic patterns
- **[/verify](/commands/verify)** - Comprehensive verification (build, types, lint, tests)

### Build & Error Fixing

Commands for resolving build errors:

- **[/build-fix](/commands/build-fix)** - Fix build and type errors incrementally
- **[/go-build](/commands/go-build)** - Fix Go build errors with minimal changes

### Refactoring & Cleanup

Commands for code maintenance:

- **[/refactor-clean](/commands/refactor-clean)** - Safely identify and remove dead code

### Continuous Learning

Commands for learning from your workflow:

- **[/learn](/commands/learn)** - Extract reusable patterns from session
- **[/learn-eval](/commands/learn-eval)** - Extract and evaluate patterns with quality gate
- **[/instinct-status](/commands/instinct-status)** - Show learned instincts
- **[/instinct-export](/commands/instinct-export)** - Export instincts to shareable format
- **[/instinct-import](/commands/instinct-import)** - Import instincts from file/URL
- **[/promote](/commands/promote)** - Promote project instincts to global
- **[/evolve](/commands/evolve)** - Cluster instincts into skills/commands/agents
- **[/projects](/commands/projects)** - List projects and instinct stats

### Skills & Automation

Commands for skill management:

- **[/skill-create](/commands/skill-create)** - Generate skills from git history analysis

### Documentation

Commands for maintaining documentation:

- **[/update-docs](/commands/update-docs)** - Sync docs with codebase sources of truth
- **[/update-codemaps](/commands/update-codemaps)** - Generate architecture codemaps

### Workflow Management

Commands for managing development workflows:

### Multi-Model Collaboration

## Usage Patterns

### New Feature Workflow

```bash
# 1. Plan the feature
/plan Add real-time notifications

# 2. Implement with TDD
/tdd

# 3. Review code quality
/code-review

# 4. Verify everything passes
/verify
```

### Bug Fix Workflow

```bash
# 1. Write failing test first
/tdd Reproduce login error

# 2. Fix the bug
# (make changes)

# 3. Verify fix
/verify
```

### Multi-Model Workflow

```bash
# 1. Collaborative planning
/multi-plan Add payment integration

# 2. Execute plan
/multi-execute .claude/plan/payment.md
```

## Command Conventions

### Arguments

Most commands accept natural language descriptions:

```bash
/plan I need to add real-time notifications
/tdd function to validate email addresses
/e2e Test the checkout flow
```

### Flags

Some commands support flags:

```bash
/verify quick              # Only build + types
/verify pre-pr             # Full checks + security
/instinct-export --domain testing
/evolve --generate         # Generate files
```

### File Paths

Commands that reference files accept paths:

```bash
/multi-execute .claude/plan/feature.md
/sessions load my-session
```
